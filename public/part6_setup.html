🚀 How to Deploy / Host the Platform

Option A: Free Online Hosting (Recommended for school use)

1
Create a free GitHub account

Go to github.com and sign up for a free account.

2
Create a new repository

Click "New Repository", name it ghsmbonjo-cs, set it to Public, click Create.

3
Upload all 6 HTML files

Click "Upload files", drag and drop all 6 part*.html files, then click "Commit changes".

4
Enable GitHub Pages

Go to Settings → Pages → Source: Deploy from branch → main → root. Click Save.

5
Rename part1_index.html to index.html

Rename the file to index.html before uploading so it becomes the default page.

6
Share the URL

Your site will be at: https://[yourusername].github.io/ghsmbonjo-cs/ — share this with students!

Option B: Run Locally (No internet needed)

Simply put all 6 HTML files in ONE folder on a computer or flash drive. Open part1_index.html in any web browser (Chrome, Firefox, Edge). No installation needed.

⚠️ For local use, student scores/logs are saved in the browser's local storage. They will be cleared if the browser data is cleared. For persistent records, use GitHub Pages.

Option C: Free Hosting on Netlify

Go to netlify.com → Sign up free → Drag and drop your folder of HTML files → Netlify instantly gives you a live URL. Fast and simple.

🔧 Admin Usage Guide

How to access the Admin Dashboard

  1. Open part1_index.html (or the hosted URL)
  2. Click "Enter Platform" on the Form 5 card
  3. Enter CIN: ADMIN2026
  4. You are logged in as Administrator
  5. In the sidebar, click "Admin Dashboard"
  6. Alternatively, open part4_structured.html — the Admin tab is visible when logged in as admin

What you can see in the Admin Dashboard

  • Stats: Total unique students, total logins, average session time, quizzes taken, passes
  • Student Lookup: Search any student by name or CIN — see their login count, total time, quiz average
  • Login History: Last 50 logins with CIN, name, date/time, and session duration
  • Quiz Results: All quiz attempts with student name, topic, score, and date
  • Export CSV: Download all quiz results as a CSV spreadsheet for your records
  • Grade Paper 2: In part4_structured.html, click "Grade" next to each part to enter student marks

✏️ How to Add/Edit Content

Add a new student

Open part1_index.html in a text editor (Notepad++, VS Code). Find the const STUDENTS = { section. Add a new line:

"116285XXX": "NEW STUDENT NAME",

Do the same in part2_platform.html, part4_structured.html, and part5_mockpapers.html.

Add new quiz questions

Open part3_quiz.html. Find const ALL_QUESTIONS = [. Add a new question object:

{id:200, topic:'Computer Hardware', source:'Your Source', q:'Question text?', opts:['A. Option','B. Option','C. Option','D. Option'], ans:0, exp:'Explanation here.', wrong:{'B':'Why B is wrong','C':'Why C is wrong','D':'Why D is wrong'}}

Change the admin password (CIN)

In part1_index.html, find const ADMIN_CIN = "ADMIN2026" and change the value. Update the same in all other part files.

Add a new mock paper

Open part5_mockpapers.html. In the const PAPERS = { section, add a new paper following the same structure as existing ones.

🛠️ Troubleshooting

Student can't login

Check that they are entering their CIN correctly (numbers only, no spaces). Verify the CIN is in the STUDENTS list in part1_index.html.

Admin dashboard shows no data

Data is stored in the browser's localStorage. Students must log in from the SAME browser on the SAME device for data to accumulate. For shared tracking, consider hosting online.

Files not linking to each other

All 6 HTML files MUST be in the same folder. The filenames must match exactly: part1_index.html, part2_platform.html, etc.

Platform looks broken on very old phones

The platform requires a modern browser (Chrome 60+, Firefox 55+, Safari 11+). Update the browser if needed.
